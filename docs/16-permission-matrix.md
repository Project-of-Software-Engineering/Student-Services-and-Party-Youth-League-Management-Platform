# 权限矩阵与数据边界说明

本文档用于验收前核查不同角色在系统中的访问边界，重点覆盖学生、教师、管理员、领导四类演示账号。

## 1. 角色说明

| 角色 | 演示账号 | 主要职责 |
| --- | --- | --- |
| 学生 | `demo.student / demo1234` | 查看本人画像、流程进度、通知、审批申请、政策问答和本人电子证明 |
| 班团骨干 | `demo.secretary / demo1234` | 协助维护本人负责班级的班团组织信息、成员概况和近期工作计划 |
| 教师 | `demo.teacher / demo1234` | 参与学生管理、审批处理、通知发布、政策维护、日志查看等教学管理工作 |
| 管理员 | `demo.admin / demo1234` | 维护学生数据、政策知识库、通知、电子证明、日志审计和系统演示数据 |
| 领导 | `demo.leader / demo1234` | 查看领导端概览，处理流转到领导节点的审批事项 |

## 3. 数据边界规则

| 数据对象 | 学生边界 | 管理角色边界 |
| --- | --- | --- |
| 学生基础信息 | 仅本人绑定学生档案 | 教师、管理员可查看和维护学生台账 |
| 学生画像 | 仅本人画像；修改需提交审核 | 教师、管理员审核画像变更并写入正式画像 |
| 审批单 | 仅本人发起和本人相关审批 | 按角色节点处理，操作记录写入审批步骤和日志 |
| 通知 | 仅命中本人标签、年级、专业、班级、政治面貌的通知 | 可发布、统计阅读情况和查看发布记录 |
| 政策文档 | 查看启用政策和问答来源 | 可维护政策元信息、正文摘要、状态和台账导出 |
| 附件 | 本人上传或本人审批关联附件 | 可查看管理范围内附件 |
| 电子证明 | 本人证明 | 可维护模板、生成、撤销和查看证明 |
| 操作日志 | 不开放 | 教师、管理员、领导可审计 |

## 4. 验收建议

1. 分别使用四类演示账号登录。
2. 按上表访问学生列表、日志、政策维护、证明管理、审批处理等高风险入口。
3. 对学生账号重点验证“不能看全量、不能导入导出、不能查看他人附件、不能管理政策和证明”。
4. 对管理账号重点验证“能导入导出、能发布通知、能维护政策、能查看日志、能审核画像变更”。
5. 对领导账号重点验证“能进入领导端并处理领导节点审批”。
6. 将异常访问、403、成功访问和导出文件结果记录到验收检查清单。

## 5. 当前限制

- 当前权限以角色级控制和服务端数据范围校验为主，尚未提供超级管理员可视化权限点配置页面。
- 当前没有自动化权限测试用例，验收阶段建议按本文档人工逐项核查。
- 细粒度字段脱敏、菜单权限、数据范围配置仍属于后续增强项。

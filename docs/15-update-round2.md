## 1. 已完成更新

- 首页 Logo 和导航字号已放大，提升首屏识别度。
- 顶部外链补齐为“微人大”“图书馆”“信息公开”，分别跳转到对应官网并在新标签页打开。

验收方法：

2. 检查首页左侧栏是否显示金色人大校徽。

### 1.2 系统内侧边栏折叠与固定

- 系统内页面侧边栏改为固定在浏览器视口左侧，页面滚动时不随内容区域错位。
- 首次进入系统内页面时侧边栏默认折叠为 64px 窄栏。
- 鼠标移入侧边栏时展开，鼠标移出时折叠。
- 学生端、管理端、领导端之间切换时，会保留当前展开或折叠状态，不再每次路由切换都直接恢复折叠态。
- 展开态 Logo 放大显示，四个功能入口左对齐并放大，当前路由保留高亮。
- 折叠态显示缩小后的 Logo 和竖排“导航”，Logo 缩小为 38px 并向左微调以适配窄栏宽度。
- 展开态底部恢复显示 `1937 / 实事求是 · 服务学生成长`，账号和“退出登录”按钮在底部紧凑排列。
- “退出登录”按钮高度压缩为 34px，避免底部区域过高。

验收方法：

1. 使用 `demo.admin / demo1234` 登录。
2. 进入管理端，检查侧边栏默认是否为 64px 折叠窄栏。
3. 将鼠标移入侧边栏，检查侧边栏是否展开并显示 Logo、导航、1937、账号和退出按钮。
4. 在展开状态下点击“学生端”或“管理端”，检查切换后侧边栏是否保持展开。
5. 将鼠标移出侧边栏，检查侧边栏是否折叠为窄栏。
6. 页面向下滚动，检查左侧栏是否始终固定在浏览器视口左侧。
7. 检查折叠态左上角 Logo 是否缩小并在窄栏内向左微调。

### 1.3 电子证明生成与下载

- 新增 `CertificateTemplate` 证明模板模型，用于维护模板名称、类型、内容模板和变量字段。
- 新增 `Certificate` 电子证明模型，用于保存证明编号、关联模板、学生信息、渲染内容和状态。
- 证明编号格式为 `{TYPE前4位}-{年份}-{4位序号}`，例如 `ENRO-2026-0001`。
- 管理端支持创建证明模板，并选择模板和学生生成电子证明。
- 管理端支持查看已生成证明列表，并可撤销未撤销证明。
- 学生端支持查看个人证明列表，并下载带编号和样式的 HTML 格式证明。
- 模板支持 `{{studentName}}`、`{{studentNo}}`、`{{grade}}`、`{{major}}`、`{{className}}`、`{{certNo}}`、`{{date}}` 等变量。

验收方法：

1. 使用 `demo.admin / demo1234` 登录管理端。
2. 找到“证明模板管理”，填写模板名称、类型编码、模板内容和变量字段。
3. 点击“创建模板”，模板列表应出现新模板。
4. 在“生成电子证明”中选择模板和学生，点击“生成证明”。
5. 页面应显示生成的证明编号。
6. 在已生成证明列表中点击“撤销”，该证明状态应变为“已撤销”。
7. 使用学生账号登录学生端，检查是否可以查看并下载个人证明。

### 1.4 管理端日志筛选、导出与对象追踪

- 新增 `GET /api/logs`，支持按 action、targetType、operatorId、startDate、endDate、page、pageSize 分页筛选日志。
- 新增 `GET /api/logs/export`，支持按当前筛选条件导出日志 Excel。
- 新增 `GET /api/logs/track?targetType=X&targetId=Y`，支持按对象追踪相关操作记录。
- 管理端日志区域从简单最近列表升级为筛选面板、分页列表和导出按钮。
- 日志分页展示当前页、总页数和总条数，并提供上一页/下一页切换。

验收方法：

1. 使用 `demo.admin / demo1234` 登录管理端。
2. 找到“操作日志”区域。
3. 输入操作类型或对象类型进行筛选。
4. 检查日志列表是否按筛选条件刷新。
5. 点击“下一页”或“上一页”，检查分页信息是否同步变化。
6. 点击“导出日志”，浏览器应下载按当前筛选条件生成的 `.xlsx` 文件。

## 2. 本机启动方式

```bash
# 1. 启动 Docker 基础设施
docker compose -f infra/docker-compose.yml up -d postgres

# 2. 后端
cd backend
cp .env.example .env  # 或确保 .env 存在
npm install
npx prisma generate
npx prisma migrate deploy
npm run prisma:seed
npm run start:dev      # 监听 :3001

# 3. 前端
cd frontend
npm install
npx vite --port 5173   # 监听 :5173，代理 /api -> :3001
```

访问 `http://localhost:5173` 即可使用。

# 部署与运维计划

## 1. 本地环境

- `frontend` 本地开发服务
- `backend` 本地 API 服务
- PostgreSQL
- Redis
- MinIO

## 2. 容器化

- 使用 Docker Compose 统一启动依赖
- 前后端分离部署
- 依赖服务独立编排

## 3. 配置管理

- 数据库连接串
- Redis 地址
- MinIO 凭据
- JWT 密钥
- 环境区分开发、测试、生产

## 4. 备份与恢复

- 数据库定期备份
- 文件对象存储定期备份
- 关键配置独立保存

## 5. 运维检查项

- 登录可用
- 接口可达
- 上传可用
- 日志可查
- 容量可控

## 6. 后续建议

- 增加 CI/CD
- 增加接口监控
- 增加错误告警
- 增加审计报表

## 7. 部署验证入口

- 面向当前工作区的实际验证步骤，见 [部署验证指南](./07-deployment-verification.md)
- 当前优先目标不是正式生产发布，而是：
  - 本机可完整启动前后端与依赖
  - 同一局域网中的另一台设备可以访问前端页面
  - 另一台设备可以完成登录、学生端访问、管理端操作验证

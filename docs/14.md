# 本轮更新与验收记录

本文档记录 2026-05-25 之后的增量更新、服务器部署状态和网站验收路径。

## 1. 已完成更新

### 1.1 通知精准推送与阅读追踪

- 管理端发布通知支持按标签、年级、专业、班级、政治面貌组合筛选接收人。
- 管理端已发布通知列表展示接收人数、已读人数、未读人数。
- 学生端通知卡片展示未读/已读状态，并支持点击“标记已读”。
- 学生端顶部统计从“我的通知”调整为“未读通知”。
- 已修复学生端模块间距，避免通知与上方模块贴在一起。

验收方法：

1. 打开 `http://10.10.0.14/`。
2. 使用 `demo.student / demo1234` 登录。
3. 进入学生端，查看“未读通知”卡片。
4. 找到“未读按钮验收通知”，点击“标记已读”。
5. 页面刷新后，该通知应显示为已读，未读数量减少。
6. 使用 `demo.admin / demo1234` 登录管理端。
7. 在“通知发布”中查看已发布通知，应能看到接收、已读、未读统计。

### 1.2 政策问答来源溯源

- 学生端政策检索现在展示问答依据来源。
- `GET /api/policies/ask` 返回 `sources`，包含来源文件、分类、版本和文件 key。
- 学生端输入关键词后，可看到“来源：文件名 | 分类 | 版本”。

验收方法：

1. 使用学生账号进入学生端。
2. 在“政策检索”输入 `党团`。
3. 检查回答下方是否出现政策来源，例如 `党员发展流程手册.md`、`团学活动考勤规定.md`。

### 1.3 领导端命名与账号修复

- 页面标题从“领导驾驶舱”调整为“领导端”。
- 首页入口从“领导审批”调整为“领导端”。
- 服务器演示账号 `demo.leader / demo1234` 已补齐并可登录。

验收方法：

1. 使用 `demo.leader / demo1234` 登录。
2. 进入领导端。
3. 页面标题应显示“领导端”，并能看到审批摘要和审批队列。

### 1.4 政策文件上传与来源绑定

- 管理端政策知识库支持上传 PDF、Word、Excel、图片、纯文本或 Markdown 附件。
- 政策知识库顶部新增醒目的“选择政策附件”上传区域，选中文件后按钮显示“上传政策附件”。
- 上传政策文件时会自动保存附件记录，并新增政策记录。
- 新增政策的 `sourceFileName` 会显示为上传文件名，学生端政策检索和问答来源可追溯到该文件。
- 未选择附件时，仍保留原来的手动新增政策记录方式。

验收方法：

1. 使用 `demo.admin / demo1234` 登录。
2. 进入管理端，找到“政策知识库”。
3. 在“政策知识库”标题下方点击“选择政策附件”，选择一个 `.pdf`、`.docx`、`.xlsx`、`.txt` 或 `.md` 文件。
4. 检查上传区域显示已选择的文件名，并确认右上角按钮变为“上传政策附件”。
5. 填写或确认标题、分类、版本。
6. 点击“上传政策附件”。
7. 页面提示“已保存”，政策列表中应出现新政策和对应来源文件名。
8. 切换到学生端，在政策检索中输入该政策标题或分类关键词。
9. 检查问答回答下方是否展示该上传文件作为来源。

### 1.5 学生导入模板下载与学生数据导出

- 管理端“学生导入”新增“下载导入模板”按钮，下载 `.xlsx` 模板。
- 导入模板表头与系统 Excel 导入字段保持一致：学号、姓名、年级、专业、班级、政治面貌、状态、简介、标签、荣誉、竞赛、实践。
- 管理端新增“导出学生数据”按钮，可导出当前学生基础信息和画像字段。
- 下载模板和导出学生数据均会写入操作日志，便于审计追踪。

验收方法：

1. 使用 `demo.admin / demo1234` 登录。
2. 进入管理端，找到“学生导入”。
3. 点击“下载导入模板”，浏览器应下载 `students-import-template.xlsx`。
4. 打开模板，检查表头是否包含学号、姓名、年级、专业、班级、政治面貌、状态、简介、标签、荣誉、竞赛、实践。
5. 点击“导出学生数据”，浏览器应下载 `students-export.xlsx`。
6. 打开导出文件，检查是否包含已有学生记录。
7. 在管理端最近日志中检查是否出现“下载学生导入模板”或“导出学生数据”记录。

### 1.6 政策编辑、停用与版本维护

- 政策文档新增启用/停用状态，默认启用。
- 管理端政策知识库可编辑标题、分类、版本和来源文件名。
- 管理端可停用政策；停用后学生端政策检索和问答默认不再返回该政策。
- 管理端可重新启用政策，也可将已有政策复制为新版再保存。
- 政策编辑、启用和停用均写入操作日志。

验收方法：

1. 使用 `demo.admin / demo1234` 登录。
2. 进入管理端，找到“政策知识库”。
3. 在任意政策卡片点击“编辑”，修改版本或来源文件名后点击“保存政策”。
4. 检查政策卡片内容是否更新，最近日志中是否出现“编辑政策”。
5. 点击该政策的“停用”，状态应变为“已停用”。
6. 切换到学生端，用该政策标题关键词检索，应不再命中该停用政策。
7. 回到管理端点击“启用”，状态应恢复为“启用中”。
8. 点击“复制新版”，表单会带入该政策信息并生成新版本号，可保存为新政策记录。
